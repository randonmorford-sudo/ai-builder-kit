# Testing and Quality Skill

Testing ensures the product remains stable as features evolve.

## Required Test Coverage

Core workflows must be verified.

Examples:

- authentication
- data creation
- data retrieval

---

## Edge Cases

Test:

- invalid input
- missing data
- unauthorized access

---

## Test Location

Tests should live near the feature.

Example:

project-service.ts  
project-service.test.ts

---

## Manual QA

Before release verify:

- main user flows work
- UI is responsive
- no console errors exist
